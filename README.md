# Livan Zhou — Research and Project Evidence Index

This repository is a structured evidence index for selected research, agent-system, machine-learning, and software-engineering work. It separates original project dates from later public-archive work and does not fabricate historical Git activity.

## Current research

### OpenCycle-Mem
**Reversible Evidence Convergence and Governed Memory for Long-Horizon Agents**

A standalone runnable repository with reversible checkpoints, provenance-aware evidence weighting, raw-memory and static-governance baselines, five controlled evidence-stream scenarios, 1,000-episode aggregate results, automated tests, and CI.

- [Standalone repository](https://github.com/haveanicedaymydear/OpenCycle-Mem)

### DTD-LRC
**Uncertainty-Gated Evidence Escalation for AI-Text Detection**

First-author manuscript submitted to UncertaiNLP 2026 at EMNLP.

- [Standalone repository](https://github.com/haveanicedaymydear/AI-Text-Cascade-Detect)

### Learning What to Learn Next
**Sparse Intervention-Calibrated Developmental Leverage for Autonomous Curricula**

Ongoing sole-author Developmental Agency research. Proposal v0.3 studies whether a low-cost, agent- and stage-specific cross-skill transfer signal, calibrated by a small number of real checkpoint interventions, can identify which currently learnable capability should receive the next training budget. The public repository contains a transparent DLS scaffold, tests, research plan, evaluation protocol, and an explicit long-term path toward an uncertainty-aware neural Counterfactual Unlock Value predictor.

- [Standalone repository](https://github.com/haveanicedaymydear/Unlock-Driven-Competence-Gap-Discovery)
- [Public proposal v0.3](https://github.com/haveanicedaymydear/Unlock-Driven-Competence-Gap-Discovery/blob/main/docs/proposal-v0.3.md)

## Evidence policy

- Original project dates and public-archive dates are kept separate.
- Numerical claims are retained only when supported by reports, logs, screenshots, datasets, or reproducible code.
- Coursework briefs, private data, credentials, restricted collaborative assets, and third-party material are not redistributed.
- A project is promoted to a standalone flagship repository only when its public release is sufficiently clear, honest, and useful to reviewers.
